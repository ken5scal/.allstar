# docs

`docs/` は一般的なドキュメント置き場であり、補助説明や運用メモを置く。人間には読みやすく保ちつつ、`.kiro/steering/` のような mandatory project memory とは分けて扱う。

## 置くもの

- 運用ガイド
- 観測・調査メモ
- 補助的な設計説明
- チームや将来の自分向けの読み物

## 置かないもの

- feature の正式な要件・設計の正本
- AI 実装セッションの rough notes
- PLAN ファイル

正式な要件や設計は `.kiro/specs/`、rough notes は `docs/ai/`、plan は `docs/ai/plan/` を使う。
