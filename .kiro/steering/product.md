# プロダクト概要

updated_at: 2026-05-10

## 目的

このリポジトリは `obsflow` の実装と運用知識をまとめる場所である。`obsflow` は RSS / X などの情報源をローカルで収集し、Obsidian Vault に Markdown-first で保存し、要約や digest を追記できる個人向けパイプラインとして扱う。

同時に、このリポジトリは AI coding agent を前提に継続開発する。コードだけでなく、要件・設計・運用知識が人間から見て信頼できる状態で残ることを重視する。

## 価値

- 情報収集結果を Obsidian 上の durable な記録として残せること
- ローカル実行を中心に、収集・要約・digest を繰り返し再現できること
- mock provider を使って外部依存なしでも開発・検証を進められること
- 人間レビュー済みの文書を project memory として保持し、別セッションの agent でも安全に再開できること

## 中核ワークフロー

- 設定ファイルと環境変数を検証してから処理を起動する
- `tick` / `run` で収集ジョブ、要約ジョブ、digest ジョブを実行する
- 1 source = 1 record で Vault ノートと Base 表示を更新する
- 失敗はログと Slack 通知で観測できるようにする
- 実装で得た知見は作業ログに残し、受け入れ済みの設計や要件に反映する

## 重要な判断基準

- 人間の信頼性を agent の速度より優先する
- ローカルで再現できることを、複雑な自動化より優先する
- 外部 I/O やコストが絡む処理は、実行範囲が見えることを重視する
- 受け入れ済みの挙動が変わるなら、コードだけでなくテストと文書も同じライフサイクルで更新する

## 非ゴール

- ここで全コマンドや全設定キーの一覧を管理しない
- 一時的な作業メモや未承認の仮説を stable な steering に混ぜない
